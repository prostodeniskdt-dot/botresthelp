# ТЗ для агента-разработчика: обновление ТТК в Telegram-боте и единое красивое оформление разделов

## 1. Задача

В существующем Telegram-боте уже есть раздел с ТТК, но данные в нём старые. Нужно обновить ТТК, используя новый файл **`ТТК Мычара  (1).xlsx`**.

Параллельная задача: привести оформление карточек во всех разделах бота к более красивому, читаемому и профессиональному виду. Сейчас карточки выглядят как простой текстовый список «поле: значение» без визуальной структуры. По скриншоту видно, что информация полезная, но подача плотная: нет разделения на смысловые блоки, заголовки не выделены, длинные поля сливаются, а карточка выглядит как выгрузка из таблицы.

Нужно сделать так, чтобы бот ощущался как аккуратная внутренняя база знаний для бара: быстро найти позицию, легко прочитать состав, понять способ приготовления, подачу, выход, заготовки и важные примечания.

---

## 2. Источник данных

Основной источник для обновления ТТК:

- файл: **`ТТК Мычара  (1).xlsx`**
- подготовленный файл для импорта: **`ttk_seed_data.json`**
- количество пользовательских карточек ТТК: **80**
- количество разделов ТТК: **8**

Использовать листы:

| Раздел в боте | Кол-во позиций | Лист Excel |
|---|---:|---|
| 🍸 Авторские коктейли | 9 | `Авторские коктейли` |
| 🥃 Классические коктейли | 21 | `Классические коктейли` |
| 🫖 Авторские чаи | 3 | `Авторские чаи` |
| 🍋 Лимонады | 3 | `Лимонады` |
| 🍷 Сезон | 1 | `Сезон` |
| 🍶 Настойки | 33 | `Настойки` |
| 🧪 Заготовки | 8 | `Заготовки` |
| ☕ Чай, кофе | 2 | `Чай, кофе` |

Не использовать как пользовательские карточки ТТК:

- `Перечень инфьюз  пф (пример)`
- `Бланк заказа (пример)`
- `Списание на проработку`
- `_Cебестоимость`

Эти листы служебные/примерные. Если в боте есть отдельный менеджерский раздел себестоимости, можно использовать `_Cебестоимость` только после отдельного согласования. В обычной библиотеке и ТТК себестоимость пользователям не показывать.

## 4. Требования к обновлению данных

### 4.1. Старые ТТК

Нужно не добавлять новые данные поверх старых, а именно обновить старую базу ТТК.

Правильная логика:

1. Найти, где в проекте хранятся текущие ТТК:
   - JSON-файл;
   - база данных;
   - Google Sheets;
   - SQLite/PostgreSQL;
   - Python/JS-словарь;
   - другой источник.
2. Сделать резервную копию старых данных.
3. Импортировать новые данные из `ttk_seed_data.json`.
4. Обновить существующие карточки по стабильному ключу:
   - `id`, если он уже есть;
   - если старых id нет, использовать связку `category + title`.
5. Если старая позиция отсутствует в новом файле:
   - не удалять физически сразу;
   - пометить как `archived=true`;
   - убрать из пользовательской выдачи.
6. После импорта в пользовательском интерфейсе должны отображаться только актуальные ТТК из нового файла.

### 4.2. Поля карточки ТТК

Каждая карточка должна поддерживать поля:

```json
{
  "id": "строковый стабильный id",
  "category_id": "id раздела",
  "category": "Название раздела",
  "title": "Название позиции",
  "ingredients": [
    {
      "amount": "количество",
      "unit": "единица измерения",
      "name": "ингредиент"
    }
  ],
  "glass": "бокал",
  "method": "метод",
  "ice": "лед",
  "garnish": "украшение",
  "prebatch": "пребэтч",
  "preparation": "технология приготовления",
  "output": "выход",
  "notes": [],
  "source_sheet": "лист Excel",
  "source_row_start": "номер строки начала",
  "source_row_end": "номер строки конца"
}
```

Пустые поля не показывать в карточке. Например, если у позиции нет приготовления, не выводить блок «Приготовление».

## 7. Красивое оформление всех разделов бота

Нужно сделать единый стиль карточек во всех разделах: ТТК, библиотека, крепкое, вино, пиво/безалко, настойки, коктейли, заготовки, словарь, аллергены и любые другие информационные карточки.

### 7.1. Проблема текущего оформления

Сейчас карточка выглядит примерно так:

```text
Louis Tollet Premier Cru Brut
Категория: Игристые
Порция/цена: бутылка / 13500 ₽
Страна: Франция
Регион/аппелласьон: Champagne Premier Cru
Сорт(а): Chardonnay, Pinot Noir, Pinot Meunier
...
```

Информация есть, но визуально она сливается. Нужно уйти от вида «текст из Excel» к виду аккуратной карточки.

### 7.2. Новый общий принцип

Каждая карточка должна строиться по схеме:

1. крупный заголовок;
2. короткая строка-контекст;
3. разделитель;
4. смысловые блоки;
5. списки с маркерами;
6. пустые поля скрываются;
7. кнопки навигации снизу.

Использовать `HTML parse_mode`, а не Markdown, чтобы стабильнее работали жирные заголовки и спецсимволы.

Все пользовательские данные перед отправкой экранировать.

## 10. Рендерер карточек

Чтобы оформление не расползалось по разным частям проекта, нужно сделать единый слой форматирования.

Желательно создать отдельный файл/модуль:

- `renderers/ttk_renderer.py`
- `renderers/library_renderer.py`
- `renderers/common.py`

Или аналогично по текущей структуре проекта.

Нужны общие функции:

```python
def render_card_title(emoji, title, subtitle=None): ...
def render_section(title, lines, emoji=None): ...
def render_bullets(items): ...
def render_ttk_card(item): ...
def render_library_card(item): ...
def escape_html(value): ...
def compact_empty_fields(data): ...
```

Главное требование: все разделы должны использовать один визуальный язык.

## 11. Ограничения Telegram

Учесть технические ограничения Telegram:

- длина одного сообщения до 4096 символов;
- если карточка длиннее 3500 символов, лучше разбить на несколько сообщений или показать короткую карточку с кнопкой `📄 Все данные`;
- HTML нужно экранировать;
- не использовать слишком длинные callback_data;
- callback_data хранить через короткие id;
- кнопки не делать шире, чем удобно читать на телефоне.

## 12. Требования к базе данных / JSON

Данные ТТК не должны быть зашиты вручную в код.

Нужно хранить их отдельно:

- JSON;
- SQLite/PostgreSQL;
- Google Sheets;
- другой источник, если уже есть в проекте.

Главное: обновление меню должно быть возможным без ручного переписывания логики бота.

Файл `ttk_seed_data.json` уже подготовлен в структуре:

```json
{
  "schema_version": "1.0",
  "source": { "...": "..." },
  "categories": [],
  "items": []
}
```

## 13. Логика импорта

Импорт должен быть идемпотентным. Повторный запуск не должен создавать дубли.

Алгоритм:

1. Прочитать `ttk_seed_data.json`.
2. Проверить наличие обязательных полей:
   - `id`
   - `category_id`
   - `category`
   - `title`
   - `ingredients`
3. Создать/обновить категории.
4. Создать/обновить позиции.
5. Старые позиции, которых нет в новом JSON, скрыть из выдачи.
6. Записать дату обновления.
7. Вывести лог:
   - сколько категорий создано;
   - сколько категорий обновлено;
   - сколько позиций создано;
   - сколько позиций обновлено;
   - сколько старых позиций скрыто;
   - какие позиции имеют пустые ингредиенты или другие проблемы.

## 15. Критерии готовности

Задача считается выполненной, если:

1. В боте отображаются новые ТТК из файла `ТТК Мычара  (1).xlsx`.
2. Всего доступно **80 позиций**.
3. В разделах отображается правильное количество позиций.
4. Старые ТТК не показываются пользователям.
5. Поиск работает по названию и ингредиентам.
6. Карточки читаются с телефона и выглядят аккуратно.
7. Все пустые поля скрываются.
8. Длинные карточки не обрезаются Telegram.
9. У каждой карточки есть кнопки возврата и поиска.
10. Оформление обновлено не только в ТТК, но и во всех информационных разделах бота.
11. Служебные листы и себестоимость не показываются обычным пользователям.
12. Есть резервная копия старой базы ТТК.
13. Повторный импорт не создаёт дубли.

### Облепиховый Кордиал

**Категория:** Заготовки  
**Источник:** лист `Заготовки`, строки 17–41

**Состав:**
- 2000 гр — С/м облепиха
- 1500 мл — Сироп настойки
- 1000 мл — Вода фильтрованная
- 4 мл — Бензоат натрия 33%

**Приготовление:**
1.  Подготовка сиропа: В небольшом сотейнике смешайте воду и сахар. Нагревайте на среднем огне, постоянно помешивая, пока сахар полностью не растворится. Доведите сироп до кипения и сразу снимите с огня. Дайте сиропу немного остыть.

2.  Смешивание: Добавьте облепиховое пюре (предварительно размороженное, если было заморожено, и протертое через мелкое сито для максимально гладкой текстуры) в теплый сахарный сироп. Тщательно перемешайте до получения полностью однородной массы.

3.  Добавление кислот: Введите лимонную кислоту и молочную кислоту. Снова хорошо перемешайте, чтобы кислоты равномерно распределились.

4.  Фильтрация (опционально): Для получения абсолютно прозрачного и гладкого кордиала, процедите смесь через мелкодисперсное сито, несколько слоев марли или кофейный фильтр.

5.  Контроль вкуса: Попробуйте кордиал. В зависимости от сладости и кислотности самой облепихи, при необходимости можно добавить еще немного сахара для баланса или чуть-чуть лимонной кислоты для усиления кислинки.

6.  Розлив и хранение: Перелейте готовый кордиал в чистые, стерилизованные бутылки. Плотно укупорьте. Храните кордиал в холодильнике. Срок хранения составит до 2-3 недель. Для увеличения срока хранения кордиал можно пастеризовать: нагреть до 75-80°C и разлить в горячие стерилизованные бутылки.

**Выход:** 4500 мл

### Малина-Смородина Коридал

**Категория:** Заготовки  
**Источник:** лист `Заготовки`, строки 42–58

**Состав:**
- 2000 мл — С/м Малина
- 2000 мл — С/м черная смородина
- 1500 мл — Сироп настойки
- 1000 гр — Вода фильтрованная
- 6 мл — Бензоат натрия 33%

**Приготовление:**
1) Смешайте пюре смородины и малины. Добавьте ~200 мл воды и подогрейте до 40–50 °C (не кипятить).

2) Введите сахар, лимонную, яблочную и молочную кислоты, соль. Размешайте до полного растворения.

3) Пробейте блендером 20–30 секунд для однородности.

4) Процедите через мелкое сито, затем через марлю/супербэг, чтобы убрать косточки и лишнюю мякоть.

5) Долейте холодной водой до точного объёма 1000 мл, ещё раз перемешайте.

6) Охладите и выдержите 6–12 часов в холодильнике для стабилизации вкуса

**Выход:** 6000 мл

### Базиликовый Кордиал

**Категория:** Заготовки  
**Источник:** лист `Заготовки`, строки 59–82

**Состав:**
- 100 гр — Базилик свежий
- 800 мл — Сироп настойки
- 200 мл — Вода фильтрованная
- 1 мл — Бензоат натрия 33%

**Приготовление:**
1.  Подготовка базилика: Тщательно промойте листья базилика и обсушите их. Отделите листья от стеблей, так как стебли могут придать горечь.

2.  Блендирование: Поместите листья базилика, холодный сахарный сироп, воду, лимонную кислоту и соль в мощный блендер.

3.  Измельчение: Перебейте смесь на высокой скорости в течение 2-3 минут, до получения максимально однородной и яркой зеленой массы. Убедитесь, что все частички базилика хорошо измельчены.

4.  Процеживание: Процедите полученную смесь через мелкоячеистое сито, прижимая твердые частицы лопаткой, чтобы извлечь максимальное количество жидкости. Для получения более прозрачного кордиала можно дополнительно процедить через марлю, сложенную в несколько слоев, или кофейный фильтр.

5.  Коррекция вкуса: Попробуйте кордиал. При необходимости можно добавить еще немного лимонной кислоты для большей кислинки или сахарного сиропа для сладости.

6.  Розлив: Перелейте готовый базиликовый кордиал в чистые, стерилизованные бутылки.

**Выход:** 1000 мл
