# Техническое задание: обновление ТТК и карточек бота
## Версия 2.1 — источник «ТТК Мычара  (2).xlsx»

## 1. Результат обновления

Новая таблица является единственным источником актуальных рецептур. Старые ТТК нельзя дописывать поверх существующей базы: требуется резервная копия, идемпотентный импорт и архивирование позиций, которых больше нет в источнике.

Фактически обнаружено **78 пользовательских карточек**:

| Раздел | Карточек | Статус |
|---|---:|---|
| 🍸 Авторские коктейли | 8 | активен |
| 🥃 Классические коктейли | 20 | активен |
| 🫖 Авторские чаи | 3 | активен |
| 🍋 Лимонады | 4 | активен |
| 🍶 Настойки | 35 | активен |
| 🧪 Заготовки | 8 | активен |
| ☕ Чай, кофе | 0 | лист пуст |
| **Итого** | **78** | |

В старом ТЗ было указано 80 карточек и раздел «Сезон». В новом Excel листа «Сезон» нет, а «Чай, кофе» не содержит карточек. Разработчик не должен создавать недостающие данные искусственно.

## 2. Файлы импорта

- `ttk_seed_data_v2.json` — нормализованная база для импорта;
- `library_seed_data_v2.json` — объединённая библиотека;
- `ТТК_Мычара_обновленная_v2.xlsx` — рабочая таблица с контрольными листами;
- этот документ — человекочитаемая спецификация и проверка.

## 3. Исключённые листы

Не публиковать обычным пользователям:

- `Перечень инфьюз  пф (пример)`;
- `Бланк заказа (пример)`;
- `Списание на проработку`;
- `_Cебестоимость`.

Себестоимость может использоваться только в отдельном менеджерском модуле с разграничением прав.

## 4. Импорт и миграция

1. Сделать резервную копию текущей базы.
2. Проверить `schema_version`.
3. Нормализовать название позиции: регистр, пробелы, `ё/е` для поиска, но не для отображения.
4. Выполнить upsert по `id`; для старой базы без ID — по `category_id + normalized_title`.
5. Позиции, которых нет в новом JSON, отметить `archived=true`; физически не удалять.
6. Повторный импорт не должен создавать дубли.
7. После импорта записать дату, имя файла, количество созданных, обновлённых, архивированных и ошибочных карточек.
8. Раздел с нулём карточек скрывать из меню либо показывать служебное сообщение «раздел пока пуст».

## 5. Схема карточки

```json
{
  "id": "ttk_...",
  "category_id": "signature_cocktails",
  "category": "Авторские коктейли",
  "title": "Базиликовый Спритц",
  "ingredients": [
    {"amount": "30", "unit": "мл", "name": "Базиликовый сироп"}
  ],
  "glass": "Wine",
  "method": "Билд",
  "ice": "Кусковой",
  "garnish": "...",
  "prebatch": "...",
  "preparation": "...",
  "output": "...",
  "notes": [],
  "source_sheet": "Авторские коктейли",
  "source_row_start": 3,
  "source_row_end": 13,
  "searchable_text": "..."
}
```

Пустые поля в сообщении не выводить. Исходные номера строк обязательны: они позволяют менеджеру быстро найти рецепт в Excel.

## 6. Визуальный язык карточек

Использовать Telegram HTML, экранировать пользовательские данные. Карточка должна выглядеть как инструкция, а не как строка из Excel.

```html
🍸 <b>Базиликовый Спритц</b>
<code>Авторские коктейли</code>

━━━━━━━━━━━━━━
🧾 <b>Состав</b>
• 30 мл — Базиликовый сироп
• 50 мл — Настойка Поко Поко

🍽 <b>Подача</b>
• Бокал: Wine
• Метод: Билд
• Лёд: Кусковой

⚙️ <b>Приготовление</b>
Показывается только при наличии текста.
```

Правила:
- короткая карточка — одним сообщением;
- длиннее 3500 символов — краткая версия и кнопка `📄 Все данные`;
- до 8–10 позиций на странице;
- кнопки: `⬅️ Назад`, `🔎 Поиск`, `📋 Все ТТК`, `🏠 Главное меню`;
- callback содержит только короткий ID;
- поиск по названию, ингредиенту, методу, бокалу, гарниру, пребэтчу и технологии.

## 7. Редактура данных

Разрешено исправлять очевидные опечатки в нормализованном тексте, не меняя рецептуру и количество. Запрещено самостоятельно:
- пересчитывать граммовки;
- додумывать отсутствующий ингредиент;
- исправлять технологию по общим знаниям;
- подменять пустое поле типовой фразой.

Любое сомнение фиксируется в диагностике импорта.

## 8. Команда проверки

`/ttk_check` должна показать:

- источник и дату импорта;
- 78 активных карточек;
- количество по разделам;
- пустые разделы;
- карточки без ингредиентов;
- дубли нормализованных названий;
- архивированные позиции;
- ошибки длины карточек.

## 9. Критерии приёмки

- В выдаче ровно 78 актуальных карточек.
- Старые рецептуры не видны пользователям.
- «Сезон» не отображается как существующий лист.
- Пустой «Чай, кофе» обработан без ошибки.
- Служебные листы и себестоимость скрыты.
- Поиск находит позиции по частичному совпадению.
- Повторный импорт не создаёт дубли.
- Карточки читаются с телефона и не обрезаются.
- Менеджер может сопоставить карточку со строками Excel.

4. Процеживание: Процедите полученную смесь через мелкоячеистое сито, прижимая твердые частицы лопаткой, чтобы извлечь максимальное количество жидкости. Для получения более прозрачного кордиала можно дополнительно процедить через марлю, сложенную в несколько слоев, или кофейный фильтр.

5. Коррекция вкуса: Попробуйте кордиал. При необходимости можно добавить еще немного лимонной кислоты для большей кислинки или сахарного сиропа для сладости.

6. Розлив: Перелейте готовый базиликовый кордиал в чистые, стерилизованные бутылки.

- Выход: 1000 мл
