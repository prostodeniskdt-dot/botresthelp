# Техническое задание: библиотека Telegram-бота бара
## Версия 2.1 — объединённая база и новая редакционная модель

## 1. Назначение

Раздел `📚 Библиотека` остаётся внутренней базой знаний для официантов, барменов и менеджеров. Карточка должна помогать сотруднику выполнить три задачи: быстро проверить факты, понятно ответить гостю и подготовиться к аттестации.

Источники разделены по приоритету:

1. `ТТК Мычара  (2).xlsx` — рецептуры, техника, подача и выход;
2. старый файл библиотеки — меню, продажи, словарь и аллергены;
3. официальные сайты производителей и профильные источники — происхождение и история вина/крепкого;
4. этикетка и спецификация поставщика — финальная проверка спорных данных.

## 2. Итоговая структура данных

Объединённый файл: `library_seed_data_v2.json`.

| Раздел | Карточек |
|---|---:|
| 🍸 Коктейли | 28 |
| 🥃 Настойки/наливки | 35 |
| 🍷 Вино | 31 |
| 🥃 Крепкий алкоголь | 41 |

| ⚠️ Аллергены | 12 |
| **Всего** | **250** |

## 3. Навигация

`📚 Библиотека → раздел → группа → позиция → карточка`

В карточке:
- `💬 Как сказать гостю`;
- `📖 История`;
- `❓ Аттестация`;
- `⚠️ Важно`;
- `🧾 Все данные`;
- `⬅️ Назад`;
- `🏠 В библиотеку`.

Поиск индексирует название, категорию, страну, регион, сорта, ингредиенты, вкус, историю, pairing, аллергены и продажную фразу.

## 4. Новая редакционная модель

Старые тексты были построены по одной формуле и звучали как автоматическая выгрузка. В версии 2.1 карточки разделены на разные уровни:

- **Факт:** страна, регион, сорт, сырьё, выдержка, технология.
- **Профиль:** аромат, вкус, структура.
- **Контекст:** конкретная история региона, сорта, производителя или метода.
- **Рекомендация:** кому предложить и с какой едой.
- **Готовая реплика:** короткая фраза, которую удобно произнести вслух.
- **Проверка:** статус достоверности и комментарий менеджеру.

Запрещённые приёмы:
- одинаковое начало каждой истории;
- пустые слова «премиальный», «уникальный», «легендарный» без объяснения;
- выдуманная история из одного названия;
- фразы «ценность напитка — в…», «ощущение праздника без тяжести» как универсальный шаблон;
- достраивание страны, сорта или выдержки без источника.

## 5. Карточка вина

```html
🥂 <b>{title}</b>
<code>{group} • {country} • {region}</code>

━━━━━━━━━━━━━━
💰 <b>Формат / цена</b>
{format} — {price}

🍇 <b>Сорт</b>
{grapes}

👅 <b>Профиль</b>
{structure}

🌿 <b>Ароматика</b>
{aroma}

🗺 <b>История и происхождение</b>
{origin_story}

🍽 <b>К чему предложить</b>
{pairing}

💬 <b>Как сказать гостю</b>
{sales_phrase}
```

Статус `Нужна сверка этикетки` виден менеджеру, но не должен превращаться в историю для гостя.

## 6. Карточка крепкого алкоголя

```html

🌾 <b>Основа</b>
{base}

👅 <b>Вкус</b>
{taste}

🏭 <b>История и технология</b>
{origin_story}

🍸 <b>Подача</b>
{best_service}

💬 <b>Как сказать гостю</b>
{sales_phrase}
```

## 7. Проверка источников

Для каждой исследованной позиции сохранять:
- `source_url`;
- `verification_status`;
- `verification_note`;
- дату последней проверки.

Если точное вино или релиз не идентифицированы:
1. не придумывать регион и историю;
2. вынести позицию в лист `Проверить_этикетки`;
3. запросить фото лицевой и контрэтикетки;
4. после подтверждения обновить JSON и выполнить повторный импорт.

## 8. Импорт

- стабильный ID;
- upsert без дублей;
- старые отсутствующие карточки архивируются;
- данные хранятся отдельно от кода;
- длинные карточки сокращаются рендерером;
- пустые поля скрываются;
- все строки экранируются для Telegram HTML;
- админский импорт пишет полный лог.

## 9. Критерии приёмки

- Новые ТТК заменили старые рецептуры.
- Вино: 31 переработанная карточка.
- Крепкий алкоголь: 41 переработанная карточка.
- У спорных позиций есть честный статус проверки.
- В карточках нет однотипных нейросетевых историй.
- История содержит конкретный факт о сорте, регионе, бренде или технологии.
- Сотрудник может использовать реплику без редактирования вслух.
- Обычный пользователь не видит себестоимость и служебные данные.
